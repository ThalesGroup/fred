Conclusion — What still needs to be done

The Swift schema migrations are partially in place. The following scripts and cleanup migrations still need to be written or confirmed before the Kea → Swift cut-over can be executed.

  • backfill · control-plane Backfill script: copy existing rows from agent → agent_instance (control-plane). Schema migration e1f2a3b4c5d6 created the new table; data is not yet moved.
  • drop · control-plane Drop migration: delete the old agent table once the backfill is verified (control-plane).
  • backfill · control-plane Backfill script: copy existing rows from session → session_metadata (control-plane). Schema migration f1a2b3c4d5e6 created the new table; data is not yet moved.
  • drop · control-plane Drop migration: delete the old session table once the backfill is verified (control-plane).
  • migration · knowledge-flow Cherry-pick migration 373dbaade980 (add standard created_at/updated_at to all tables) from knowledge-flow-backend into Swift — affects metadata, resource, sched_workflow_tasks, and tag.
  • migration · control-plane Cherry-pick migration ee06c110f46c (add standard created_at/updated_at to all tables) from control-plane-backend into Swift — affects teammetadata and users.
  • backfill · fred-runtime Backfill script: copy existing session_history rows from the Kea agentic backend into the Swift fred-runtime database. Migration a1e2f3c4d5b6 created the new table; data is not yet moved. Also: clarify whether multiple pod versions share one DB or each use their own (isolation model TBD).
  • drop · fred-runtime Drop migration: delete the old session table in fred-runtime once the session_history backfill is verified.
  • backfill · control-plane Backfill script (if applicable): move existing session_attachments rows into the metadata table if that is the confirmed migration target. Requires design decision first.
  • drop · control-plane Drop migrations: delete tables feedbacks, mcp-server, session_attachments, and tasks from control-plane (all deleted in Swift). Confirm intent and data-archival policy before dropping.
