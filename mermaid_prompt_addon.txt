When you include Mermaid diagrams, follow these rules strictly so the diagram always parses:

1) Output Mermaid inside fenced code blocks only:
```mermaid
...
```

2) For flowcharts, ALWAYS quote node labels:
- Use: ID["Label text"]
- Never use: ID[Label text] when text has spaces, punctuation, HTML, or parentheses.

3) If a label needs a line break, keep it inside quotes:
- Good: A["Web App<br/>Europe"]
- Avoid unquoted labels with <br/>, (, ), [, ], :, ;, ,.

4) Keep node IDs simple and stable:
- Letters/numbers/underscore only (e.g., API_1, LB2, DB_MAIN).

5) Use only valid edge syntax:
- A --> B
- A -->|text| B

6) Avoid advanced syntax unless necessary:
- No custom directives/macros.
- Keep linkStyle indexes minimal and valid.

7) Before returning, self-check:
- Every opened subgraph is closed.
- Every node label with special chars is quoted.
- Diagram is Mermaid v11 compatible.

If you are unsure, return a simpler valid Mermaid diagram instead of a complex risky one.

Optional extra instruction (very useful):
After the Mermaid block, add one short line: "Mermaid validity check: passed".
